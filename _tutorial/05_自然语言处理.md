# 05 - 自然语言处理（NLP）

> 让AI理解、处理和生成人类语言的技术

## 5.1 什么是自然语言处理？

### 定义

**自然语言处理（NLP）= 让计算机理解、处理和生成人类语言**

```
人类语言 vs 编程语言:

编程语言:
├── 语法严格
├── 无歧义
├── 结构规整
└── 计算机直接能执行

自然语言:
├── 语法灵活
├── 充满歧义
├── 不规则
├── 依赖上下文
└── 计算机需要专门学习才能理解

例:
"苹果" → 水果？还是公司？
"我一把把把手把住了" → 什么意思？
```

### NLP为什么难？

```
挑战1: 歧义性
"我喜欢苹果"
→ 喜欢水果？还是喜欢iPhone？

挑战2: 上下文依赖
"他很好"
→ 谁？什么好？需要前文

挑战3: 语言的灵活性
同样的意思，无数种说法:
- "你好"
- "您好"
- "哈喽"
- "Hi"
- "早上好"

挑战4: 常识推理
"小明把杯子打翻了，水洒了"
→ 需要知道：杯子装水、重力、液体流动...

挑战5: 隐喻和情感
"他的心如死灰"
→ 不是真的心脏是灰的
```

## 5.2 NLP的主要任务

### 任务分类

```
NLP任务:
├── 理解类（NLU）
│   ├── 文本分类
│   ├── 命名实体识别
│   ├── 情感分析
│   └── 语义相似度
│
├── 生成类（NLG）
│   ├── 机器翻译
│   ├── 文本摘要
│   ├── 对话生成
│   └── 文章创作
│
└── 转换类
    ├── 语音识别（语音→文本）
    ├── 语音合成（文本→语音）
    └── OCR（图像→文本）
```

### 详细说明

**1. 文本分类（Text Classification）**
```
输入: 一段文本
输出: 类别标签

应用:
├── 垃圾邮件识别
├── 新闻分类（体育/财经/科技）
├── 情感分析（正面/负面）
└── 意图识别（问天气/查路线）

例:
"这部电影太精彩了！" → 正面情感
"完全浪费时间" → 负面情感
```

**2. 命名实体识别（NER）**
```
输入: 文本
输出: 实体及其类型

实体类型:
├── PER: 人名（张三、马云）
├── ORG: 组织（清华大学、腾讯）
├── LOC: 地点（北京、长江）
├── TIME: 时间（2024年、明天）
└── ...

例:
"马云在杭州创立了阿里巴巴"
→ [马云-PER] 在 [杭州-LOC] 创立了 [阿里巴巴-ORG]
```

**3. 机器翻译（Machine Translation）**
```
输入: 源语言文本
输出: 目标语言文本

类型:
├── 规则翻译（早期，基于语法规则）
├── 统计翻译（基于双语语料统计）
└── 神经翻译（当前主流，用神经网络）

例:
"Hello world" → "你好，世界"
```

**4. 问答系统（Question Answering）**
```
输入: 问题（+ 可选的上下文）
输出: 答案

类型:
├── 抽取式: 从文本中抽取答案片段
├── 生成式: 自己组织语言生成答案
└── 知识库问答: 从结构化知识中检索

例:
问: "谁创立了阿里巴巴？"
文: "马云于1999年创立了阿里巴巴"
答: "马云"
```

**5. 文本摘要（Text Summarization）**
```
输入: 长文本
输出: 简短摘要

类型:
├── 抽取式: 从原文抽取关键句子
└── 生成式: 理解后重新组织语言生成

例:
原文: 500字的新闻报道
摘要: 50字的核心内容
```

## 5.3 文本表示：如何让计算机理解文字

### 问题：计算机只认识数字

```
文本: "猫坐在垫子上"

计算机不能直接理解，需要转成数字表示
```

### 方法一：One-Hot 编码

```
词表: ["猫", "坐", "在", "垫子", "上"]

"猫" = [1, 0, 0, 0, 0]
"坐" = [0, 1, 0, 0, 0]
"在" = [0, 0, 1, 0, 0]
...

问题:
- 维度灾难（词表几万到几十万）
- 所有词距离都一样（无法表达相似性）
- 稀疏表示（大部分为0）
```

### 方法二：词嵌入（Word Embedding）

**核心思想**: 用稠密向量表示词，语义相似的词向量距离近

```
词 → 低维向量（通常50-300维）

例:
"猫" = [0.2, -0.5, 0.8, ...]
"狗" = [0.3, -0.4, 0.7, ...]  ← 距离近
"汽车" = [-0.8, 0.9, 0.1, ...] ← 距离远

神奇性质:
"国王" - "男人" + "女人" ≈ "女王"
"北京" - "中国" + "日本" ≈ "东京"
```

### Word2Vec（2013年）

```
训练目标: 预测上下文

CBOW: 用周围词预测中心词
Skip-gram: 用中心词预测周围词

例（Skip-gram）:
输入: "猫"
预测: "坐", "在", "垫子", "上"

通过大量文本训练，
词的向量自然就包含了语义信息
```

### 更好的嵌入：上下文相关

**问题**: Word2Vec一个词只有一个向量
```
"bank" 在 Word2Vec 中只有一个向量，
但可能是"银行"也可能是"河岸"
```

**解决方案**:
```
ELMo (2018):
├── 双向LSTM
├── 词的表示依赖整个句子
└── 一词多义问题解决

BERT (2018):
├── Transformer编码器
├── 真正的上下文相关
└── 成为NLP的基础
```

## 5.4 Transformer：NLP的革命

### 出现之前：RNN的问题

```
RNN处理序列:
"我 喜欢 深度 学习"
↓
时间步1 → 时间步2 → 时间步3 → 时间步4
  ↓          ↓          ↓          ↓
"我"       "喜欢"     "深度"     "学习"

问题:
1. 无法并行计算（必须等前一个算完）
2. 长距离依赖困难（前面信息传到后面会丢失）
```

### Transformer的核心创新

**Self-Attention（自注意力）**
```
核心思想:
每个词都要看看其他所有词，
决定关注谁、关注多少

过程:
1. 每个词生成Query、Key、Value三个向量
2. Query和所有的Key计算相似度（注意力权重）
3. 用权重对Value加权求和，得到新表示

优点:
- 可以并行计算
- 长距离依赖直接连接
- 可解释性强（能看到模型关注哪里）
```

### 注意力计算示例

```
句子: "我喜欢机器学习"

计算"喜欢"的表示:
        Q(喜欢) · K(我)      → 注意力权重
        Q(喜欢) · K(喜欢)    → 注意力权重
        Q(喜欢) · K(机器)    → 注意力权重
        Q(喜欢) · K(学习)    → 注意力权重
        ↓
    用权重对V加权求和
        ↓
    "喜欢"的新表示（包含上下文）
```

### Multi-Head Attention

```
问题: 一种注意力可能不够

解决: 多头注意力
- 并行计算多组注意力
- 每组关注不同方面
- 最后拼接起来

例:
Head 1: 关注语法关系
Head 2: 关注语义关系
Head 3: 关注指代关系
...
```

### Transformer完整结构

```
Encoder（编码器）:
输入 → Embedding + 位置编码
  ↓
[Multi-Head Attention]
[Add & Norm] ← 残差连接+层归一化
  ↓
[Feed Forward]
[Add & Norm]
  ↓
输出（上下文表示）

Decoder（解码器）:
类似Encoder，但多一个Cross-Attention
关注Encoder的输出
```

## 5.5 预训练语言模型

### 为什么要预训练？

```
问题:
- NLP任务需要大量标注数据
- 标注数据 expensive
- 每个任务都从头训练浪费

解决:
1. 用海量无标注文本预训练（学习语言规律）
2. 用少量标注数据微调（适应具体任务）

效果:
- 小数据也能训练好模型
- 一个预训练模型适配多种任务
```

### BERT（2018）

```
架构: Transformer Encoder
创新: 双向上下文理解

预训练任务:
1. MLM (Masked Language Model)
   遮盖15%的词，让模型预测
   "我 [MASK] 机器学习" → "喜欢"

2. NSP (Next Sentence Prediction)
   判断两句话是否是连续的

微调:
- 分类任务: [CLS]token接分类器
- 序列标注: 每个token接分类器
```

### GPT系列

```
GPT-1 (2018):
├── Transformer Decoder
├── 单向语言模型
└── 生成能力

GPT-2 (2019):
├── 15亿参数
├── Zero-shot能力
└── 不开放（怕滥用）

GPT-3 (2020):
├── 1750亿参数
├── Few-shot能力惊人
└── API开放

GPT-4 (2023):
├── 多模态（图文）
├── 推理能力大幅提升
└── 当前顶级模型

核心思想:
给定前文，预测下一个词
→ 自然形成生成能力
```

### 预训练模型家族

| 模型 | 架构 | 特点 | 适合 |
|------|------|------|------|
| **BERT** | Encoder | 双向理解 | 分类、NER |
| **GPT** | Decoder | 单向生成 | 生成、对话 |
| **T5** | Encoder-Decoder | 统一框架 | 翻译、摘要 |
| **RoBERTa** | Encoder | BERT优化版 | 分类任务 |
| **ALBERT** | Encoder | 轻量BERT | 资源受限 |

## 5.6 NLP实践指南

### 开发流程

```
1. 数据准备
   ├── 收集文本数据
   ├── 清洗（去噪、去重）
   └── 标注（如果需要）

2. 文本预处理
   ├── Tokenization（分词）
   ├── 构建词表
   └── 转成数字ID

3. 选择模型
   ├── 简单任务: 传统ML（TF-IDF+SVM）
   ├── 一般任务: 预训练模型微调
   └── 生成任务: GPT类模型

4. 训练
   ├── 加载预训练权重
   ├── 冻结部分层（可选）
   └── 微调训练

5. 评估部署
```

### 轻量级NLP（适合你的电脑）

```
词向量模型:
├── Word2Vec（训练快）
├── GloVe（稳定）
└── FastText（支持子词，OOV友好）

预训练语言模型:
├── DistilBERT（BERT的蒸馏版，小40%）
├── TinyBERT（更小）
├── MobileBERT（移动端优化）
└── ALBERT（参数共享，轻量）

实际建议:
- 文本分类: DistilBERT足够好
- 特征提取: 用预训练Embedding
- 避免: 大模型微调（需要大显存）
```

## 5.7 动手试试

### 实验1：词向量运算

```python
import gensim.downloader as api

# 加载预训练词向量（约100MB）
word_vectors = api.load("glove-wiki-gigaword-100")

# 找最相似的词
similar = word_vectors.most_similar('computer', topn=5)
print("与'computer'最相似的词:")
for word, score in similar:
    print(f"  {word}: {score:.4f}")

# 类比推理: king - man + woman = ?
result = word_vectors.most_similar(
    positive=['king', 'woman'],
    negative=['man'],
    topn=1
)
print(f"\nking - man + woman = {result[0][0]}")

# 计算相似度
sim = word_vectors.similarity('cat', 'dog')
print(f"\n'cat'和'dog'的相似度: {sim:.4f}")
```

### 实验2：用Transformers做文本分类

```python
from transformers import pipeline

# 加载预训练情感分析模型
classifier = pipeline('sentiment-analysis',
                     model='distilbert-base-uncased-finetuned-sst-2-english')

# 预测
texts = [
    "I love this product!",
    "This is terrible.",
    "It's okay, nothing special."
]

results = classifier(texts)
for text, result in zip(texts, results):
    print(f"文本: {text}")
    print(f"情感: {result['label']}, 置信度: {result['score']:.4f}\n")
```

### 实验3：简单的文本生成

```python
from transformers import GPT2LMHeadModel, GPT2Tokenizer

# 加载小型GPT-2（约500MB）
tokenizer = GPT2Tokenizer.from_pretrained('gpt2')
model = GPT2LMHeadModel.from_pretrained('gpt2')

# 生成文本
prompt = "Once upon a time"
inputs = tokenizer.encode(prompt, return_tensors='pt')

outputs = model.generate(
    inputs,
    max_length=50,
    num_return_sequences=1,
    temperature=0.7,
    do_sample=True
)

generated = tokenizer.decode(outputs[0], skip_special_tokens=True)
print(generated)
```

## 本章小结

**核心概念**:
1. NLP让AI理解人类语言，充满挑战

- ✅ 轻量级模型（如DistilBERT）适合个人电脑
- ✅ 生成任务和分类任务需要不同架构
