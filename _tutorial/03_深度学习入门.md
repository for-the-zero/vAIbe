## 3.1 从机器学习到深度学习

### 什么是深度学习？

**深度学习 = 使用深层神经网络的机器学习**

```
传统机器学习:
数据 → 人工设计特征 → 简单模型 → 结果

深度学习:
数据 → 神经网络自动学特征 → 结果

关键区别: 深度学习能自动学习特征表示
```

### 为什么叫"深度"？

```
深度 = 神经网络的层数很多

浅层网络: 1-2层（如传统MLP）
深层网络: 几十到几百层（现代深度学习）

层数增加 → 表达能力增强 → 能学习更复杂的模式
```

### 深度学习的优势

```
1. 自动特征学习
   不需要人工设计特征
   
2. 端到端学习
   输入直接到输出，中间自动处理
   
3. 表现优异
   图像、语音、NLP等任务都是最先进水平
   
4. 可扩展性好
   数据越多、模型越大，效果通常越好
```

## 3.2 神经网络基础

### 生物神经元的启发

```
生物神经元:
树突(接收信号) → 细胞体(处理) → 轴突(输出)

人工神经元:
输入 → 加权求和 → 激活函数 → 输出
```

### 人工神经元详解

```
        x₁ ──w₁──┐
        x₂ ──w₂──┼──→ Σ ──→ 激活函数 ──→ 输出
        x₃ ──w₃──┤      ↑
        ...      └── b ─┘
        
x = 输入（上一层传来的信号）
w = 权重（连接强度，可学习）
b = 偏置（阈值调整，可学习）
Σ = 加权求和: z = w₁x₁ + w₂x₂ + ... + b
```

### 激活函数：为什么要非线性？

**问题**: 如果没有激活函数，多层网络 = 单层网络

```
举例:
层1: y = w₁x + b₁
层2: z = w₂y + b₂ = w₂(w₁x + b₁) + b₂ = (w₂w₁)x + (w₂b₁+b₂)

还是线性函数！多层没有意义
```

**解决方案**: 加入非线性激活函数

### 常用激活函数

| 函数 | 图像 | 公式 | 特点 |
|------|------|------|------|
| **Sigmoid** | S型曲线 | 1/(1+e⁻ˣ) | 输出0-1，适合概率 |
| **Tanh** | 对称S型 | (eˣ-e⁻ˣ)/(eˣ+e⁻ˣ) | 输出-1~1，零中心化 |
| **ReLU** | 直角转折 | max(0,x) | 简单高效，默认选择 |
| **Leaky ReLU** | 带斜率的转折 | max(0.01x,x) | 解决ReLU死亡问题 |
| **Softmax** | 多类别概率 | eˣⁱ/Σeˣʲ | 多分类输出层 |

**为什么ReLU最常用？**
```
优点:
1. 计算简单（只有比较操作）
2. 缓解梯度消失（正区间梯度为1）
3. 稀疏激活（负值变0，计算量少）

缺点:
- 负值永远为0（"死亡ReLU"）
→ 用Leaky ReLU改进
```

## 3.3 神经网络的结构

### 多层感知机（MLP）

```
输入层    隐藏层1    隐藏层2    输出层

○ ────→ ○ ────→ ○ ────→ ○
○ ────→ ○ ────→ ○ ────→ ○
○ ────→ ○ ────→ ○
○ ────→ ○

每个圆圈 = 一个神经元
每条线 = 一个连接（带权重）
```

**层的作用**:
```
输入层: 接收原始数据
隐藏层: 提取特征，学习表示
输出层: 产生最终预测

深度 = 隐藏层数 + 1
宽度 = 每层的神经元数
```

### 前向传播：数据如何流动

```
输入 → 第1层计算 → 激活 → 第2层计算 → 激活 → ... → 输出

每一层的计算:
1. 线性变换: z = W·x + b
2. 激活函数: a = f(z)
3. 传递给下一层
```

### 网络容量

```
参数数量 = 层数 × 每层神经元数 × 连接数

更多参数 = 更强的表达能力 = 更容易过拟合

设计原则:
├── 任务简单 → 小网络
├── 任务复杂 → 大网络 + 正则化
└── 数据少 → 小网络
```

## 3.4 反向传播：如何训练神经网络

### 核心问题

```
我们有:
- 网络结构（几层、每层多少神经元）
- 训练数据（输入+正确答案）

我们需要:
- 找到最优的权重W和偏置b

方法:
梯度下降 + 反向传播
```

### 反向传播的思想

```
类比：工厂生产出了次品，要追责到每个环节

输出层误差大 → 最后一层工人调整 → 告诉前一层
→ 前一层工人调整 → 告诉更前一层
→ ... → 直到第一层

反向传播 = 从输出往输入，逐层计算梯度
```

### 链式法则

```
如果 z = f(y), y = g(x)
那么 dz/dx = dz/dy × dy/dx

在神经网络中:
损失函数 ← 第3层 ← 第2层 ← 第1层 ← 输入

∂Loss/∂W₁ = ∂Loss/∂a₃ × ∂a₃/∂z₃ × ∂z₃/∂a₂ × ... × ∂z₁/∂W₁

复杂，但计算机能自动算！
```

### 训练流程总结

```
1. 前向传播
   输入数据 → 层层计算 → 得到预测
   
2. 计算损失
   比较预测和真实值 → 算出差多少
   
3. 反向传播
   从输出往回算 → 每层梯度是多少
   
4. 参数更新
   权重 = 权重 - 学习率 × 梯度
   
5. 重复1-4
   直到收敛
```

## 3.5 优化器：更聪明的梯度下降

### 基础SGD的问题

```
普通梯度下降:
每次用全部数据算梯度 → 太慢
每次用一个样本算梯度 → 太不稳定

解决: 小批量梯度下降(Mini-batch)
每次用一小批(如32个)样本算梯度
```

### Momentum（动量）

```
思想: 像滚雪球，有惯性

如果连续几次都往一个方向走，
那就加速往那个方向走

好处:
- 加速收敛
- 减少震荡
- 逃离局部最优

类比: 小球滚下山，有速度，不会卡在坑里
```

### Adam（自适应学习率）

```
目前最常用的优化器

特点:
- 给每个参数不同的学习率
- 梯度大的参数，学习率小一点
- 梯度小的参数，学习率大一点
- 结合了Momentum和RMSprop的优点

默认参数通常就很好:
lr=0.001, betas=(0.9, 0.999)
```

### 优化器对比

| 优化器 | 优点 | 缺点 | 适用 |
|--------|------|------|------|
| **SGD** | 简单、泛化好 | 慢、需调学习率 | 大规模数据 |
| **Momentum** | 加速收敛 | 需调参数 | CV常用 |
| **Adam** | 自适应、快 | 可能泛化稍差 | 默认首选 |
| **AdamW** | 解耦权重衰减 | - | Transformer |

## 3.6 正则化：防止过拟合

### Dropout（随机失活）

```
训练时:
每次前向传播，随机"关闭"一些神经元（输出设0）
通常关闭概率 p=0.5

测试时:
用全部神经元，但输出乘以p

原理:
强迫网络不依赖任何一个神经元，
必须多个神经元合作才能完成任务，
提高鲁棒性

类比:
公司不依赖任何一个员工，
大家都能互相backup
```

### Batch Normalization（批归一化）

```
问题:
网络深层，输入分布会变化（Internal Covariate Shift）
导致训练困难

解决:
对每个batch的数据做归一化
x' = (x - batch_mean) / batch_std

好处:
- 训练更稳定
- 可以用更大学习率
- 轻微正则化效果
- 减少对初始化的依赖
```

### 其他正则化方法

```
L1/L2正则化:
在损失函数中加入权重的惩罚项
L1: 让权重稀疏（很多变0）
L2: 让权重小但不为0

Early Stopping:
验证集性能不提升就停止训练

Data Augmentation:
增加训练数据的多样性
```

## 3.7 深度学习框架

### 为什么需要框架？

```
不用框架，你要自己写:
- 反向传播算法
- GPU加速
- 各种层的前向/反向实现
- 优化器
- 数据加载
- ...

用框架，你只需要:
- 定义网络结构
- 定义损失函数
- 选择优化器
- 写训练循环
```

### PyTorch（推荐入门）

```
特点:
- 动态图，调试方便
- Pythonic，直观易懂
- 社区活跃，教程丰富
- 研究首选

核心概念:
├── Tensor: 多维数组，可以在GPU上计算
├── Module: 神经网络模块
├── Autograd: 自动求导
└── DataLoader: 数据加载
```

### TensorFlow/Keras

```
特点:
- 静态图，部署方便
- Keras高层API简单易用
- 工业部署成熟
- 谷歌支持

适合:
- 生产环境部署
- 移动端/嵌入式
```

### 框架选择建议

| 场景 | 推荐框架 |
|------|----------|
| 学习/研究 | PyTorch |
| 快速实验 | PyTorch Lightning |
| 生产部署 | TensorFlow/ONNX |
| 大模型训练 | PyTorch + DeepSpeed |
| 边缘部署 | TensorFlow Lite/ONNX |

## 3.8 动手试试

### 实验1：用PyTorch搭建神经网络

```python
import torch
import torch.nn as nn
import torch.optim as optim

# 1. 定义网络结构
class SimpleNet(nn.Module):
    def __init__(self):
        super(SimpleNet, self).__init__()
        self.fc1 = nn.Linear(784, 256)  # 输入784(28x28图像)，输出256
        self.fc2 = nn.Linear(256, 128)  # 隐藏层
        self.fc3 = nn.Linear(128, 10)   # 输出10(0-9数字分类)
        self.relu = nn.ReLU()
        self.dropout = nn.Dropout(0.2)
    
    def forward(self, x):
        x = x.view(-1, 784)  # 展平
        x = self.relu(self.fc1(x))
        x = self.dropout(x)
        x = self.relu(self.fc2(x))
        x = self.fc3(x)
        return x

# 2. 创建模型、损失函数、优化器
model = SimpleNet()
criterion = nn.CrossEntropyLoss()
optimizer = optim.Adam(model.parameters(), lr=0.001)

# 3. 训练循环（简化版）
for epoch in range(10):
    for data, target in train_loader:
        # 前向传播
        output = model(data)
        loss = criterion(output, target)
        
        # 反向传播
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
    
    print(f'Epoch {epoch}, Loss: {loss.item():.4f}')
```

### 实验3：理解层数的影响

```python
# 比较不同深度的网络
shallow = nn.Sequential(
    nn.Linear(784, 10)
)

deep = nn.Sequential(
    nn.Linear(784, 256),
    nn.ReLU(),
    nn.Linear(256, 128),
    nn.ReLU(),
    nn.Linear(128, 10)
)

# 参数量对比
shallow_params = sum(p.numel() for p in shallow.parameters())
deep_params = sum(p.numel() for p in deep.parameters())

print(f"浅层网络参数: {shallow_params}")
print(f"深层网络参数: {deep_params}")

# 深层网络参数更多，表达能力更强
```

## 本章小结

**核心概念**:
1. 深度学习 = 深层神经网络
2. 神经网络由神经元组成，每个神经元做加权求和+激活
3. 激活函数引入非线性，让网络能学习复杂模式
4. 反向传播从输出往输入计算梯度，更新权重
5. 正则化防止过拟合

**关键认知**:
- ✅ 深度网络能自动学习特征
- ✅ 框架让我们不用自己实现反向传播
- ✅ 优化器让训练更快更稳定
- ✅ 层数和参数要适中，防止过拟合
