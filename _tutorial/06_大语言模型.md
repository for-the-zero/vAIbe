# 06 - 大语言模型（LLM）

> ChatGPT背后的技术原理，当前AI最热门的领域

---

## 6.1 什么是大语言模型？

### 定义

**大语言模型（LLM）= 参数量巨大的语言模型（通常10亿参数以上）**

```
规模对比:
BERT-Base:    1.1亿参数
GPT-2:        15亿参数
GPT-3:        1750亿参数
GPT-4:        估计1万亿+
Llama 2 70B:  700亿参数

"大"带来质变:
- 涌现能力（Emergent Abilities）
- 上下文学习（In-Context Learning）
- 推理能力（Chain of Thought）
```

### 核心能力

```
1. 文本生成
   给定开头，续写文章、故事、代码

2. 问答对话
   回答知识性问题，进行多轮对话

3. 文本理解
   摘要、翻译、情感分析

4. 推理能力
   数学计算、逻辑推理、代码调试

5. 知识检索
   存储大量训练数据中的知识
```

### 涌现能力（Emergent Abilities）

```
小模型不具备，大模型突然出现的能力:

├── 上下文学习（Few-shot Learning）
│   给几个例子，就能学会新任务
│   例: 给3个翻译例子，就能翻译新句子
│
├── 思维链（Chain of Thought）
│   把推理过程写出来，答案更准确
│   例: "2+3=5，所以答案是5"
│
├── 指令遵循（Instruction Following）
│   理解自然语言指令并执行
│   例: "用Python写一个冒泡排序"
│
└── 代码生成
    理解需求，生成可运行代码
```

## 6.2 LLM的架构

### Decoder-Only架构

```
当前主流LLM都采用Decoder-only架构:
├── GPT系列
├── Llama系列
├── ChatGLM
├── Qwen
└── ...

原因:
- 适合生成任务
- 训练效率高
- 推理可以KV Cache优化
```

### 核心组件

**1. Tokenization（分词）**
```
文本 → Token序列

方法:
├── BPE (Byte Pair Encoding)
│   GPT系列使用
│   从字符开始，合并最常见的pair
│
└── SentencePiece
    很多开源模型使用
    统一处理多语言

例:
"Hello world" → ["Hello", " world"]
"机器学习" → ["机器", "学习"] 或 ["机", "器", "学", "习"]

注意:
- 词表大小: 通常3万-10万
- Token不等于词，可能是子词
- 英文1词≈1.3 tokens，中文1字≈1.5 tokens
```

**2. Embedding层**
```
Token ID → 向量表示

组成:
├── Token Embedding: 词本身的含义
└── Positional Encoding: 位置信息

现代方法:
使用旋转位置编码(RoPE)或ALiBi，
替代传统的正弦位置编码
```

**3. Transformer Decoder层**
```
堆叠多层（通常12-96层）:

每层包含:
├── Masked Multi-Head Self-Attention
│   只能看当前位置及之前的token
│   用因果掩码实现
│
├── Feed-Forward Network (FFN)
│   通常用SwiGLU激活
│   隐藏层维度是输入的4倍
│
└── LayerNorm + Residual Connection

参数量估算:
├── Attention: 4 × hidden_size²
├── FFN: 8 × hidden_size²
└── 每层约: 12 × hidden_size²
```

**4. 输出层**
```
最后一个隐藏层 → Linear → Softmax

输出: 词表大小的概率分布
表示下一个token预测概率

例:
输入: "今天天气"
输出概率:
  "很好": 0.3
  "不错": 0.25
  "晴朗": 0.2
  ...
```

### 自回归生成

```
生成过程:
输入: "今天"
预测: "天气" (概率最高)
输入: "今天天气"
预测: "很好"
输入: "今天天气很好"
预测: "。"
...

特点:
- 逐个token生成
- 每个新token依赖之前所有token
- 可以设置最大长度
```

## 6.3 LLM的训练

### 训练三阶段

```
阶段1: 预训练（Pre-training）
├── 数据: 海量互联网文本（TB级）
├── 任务: 下一token预测
├── 成本: 百万美元级
├── 时间: 数月
└── 产出: Base模型（懂语言但不懂对话）

阶段2: 监督微调（SFT）
├── 数据: 人工编写的对话数据
├── 任务: 学习对话格式
├── 成本: 数十万美元
├── 时间: 数天到数周
└── 产出: 能对话的模型

阶段3: 人类反馈强化学习（RLHF）
├── 数据: 人类对回答的偏好排序
├── 任务: 学习人类偏好
├── 成本: 数十万美元
├── 时间: 数天到数周
└── 产出: 对齐人类价值观的模型
```

### 预训练细节

**数据组成**:
```
Web数据:
├── Common Crawl（网页抓取）
├── Wikipedia（百科）
├── Books（书籍）
├── Code（代码）
└── ...

数据处理:
├── 去重
├── 过滤低质量内容
├── 隐私脱敏
└── Tokenization
```

**训练配置**:
```
Batch Size: 通常很大（百万级tokens）
学习率: 预热+衰减，峰值约1e-4到3e-4
优化器: AdamW
精度: 混合精度（FP16/BF16）
并行: 数据并行+模型并行+流水并行
```

### 监督微调（SFT）

**数据格式**:
```json
{
  "messages": [
    {"role": "system", "content": "你是 helpful assistant"},
    {"role": "user", "content": "你好"},
    {"role": "assistant", "content": "你好！有什么可以帮助你的吗？"}
  ]
}
```

**训练目标**:
```
只预测assistant的内容，
user和system的内容作为上下文

损失函数: 交叉熵
```

### RLHF详解

**第一步：训练奖励模型（Reward Model）**
```
数据格式:
Prompt + [回答A, 回答B] + 人类偏好(A>B或B>A)

训练目标:
让奖励模型学会预测人类偏好
奖励(Prompt, 回答A) > 奖励(Prompt, 回答B)
```

**第二步：用PPO优化策略**
```
目标:
- 最大化奖励模型打分
- 同时不要太偏离原模型（KL散度约束）

Loss = -奖励分数 + β×KL(当前模型||参考模型)

为什么需要KL约束？
防止模型为了高分而"作弊"，
生成奖励模型喜欢但人类不喜欢的内容
```

**替代方案：DPO（直接偏好优化）**
```
优点:
- 不需要显式训练奖励模型
- 不需要强化学习
- 训练更稳定
- 效果更好

原理:
直接用偏好数据优化语言模型
```

## 6.4 LLM的推理

### 推理过程

```
输入: "中国的首都是"

Step 1: Tokenize
→ ["中国", "的", "首都", "是"]

Step 2: 前向传播
→ 得到每个token的隐藏状态

Step 3: 预测下一个token
→ "北京": 0.9, "上海": 0.05, ...

Step 4: 采样（或贪心选择）
→ 选择"北京"

Step 5: 拼接到输入，重复2-4
→ 输入: "中国的首都是北京"
→ 预测下一个: "。": 0.95

直到生成结束符或达到最大长度
```

### 采样策略

**Temperature（温度）**
```
作用: 控制随机性

T→0: 接近贪心，确定性输出
T=1: 按概率采样
T>1: 更随机，有创造性

公式: softmax(logits / T)

选择:
- 事实性问题: T=0.1-0.3（准确）
- 创意写作: T=0.7-1.0（多样）
```

**Top-k采样**
```
只从概率最高的k个词中选

例:
全部词表: 5万个
Top-k=50: 只从概率最高的50个词采样

优点: 过滤掉明显不对的词
缺点: k是固定的，有时太多有时太少
```

**Top-p采样（Nucleus Sampling）**
```
从累积概率达到p的最小集合中采样

例:
p=0.9，从高到低累加概率，
可能前5个词就达到0.9，
也可能需要前100个词

优点: 动态调整候选词数量
```

**重复惩罚**
```
问题: 模型容易重复之前说过的话

解决: 对已经出现过的词降低概率
penalty = 1.0-2.0

注意: 惩罚太高会生成奇怪的内容
```

### KV Cache优化

**问题**:
```
生成第N个token时，
需要计算前面所有token的K和V，
但前面token的K/V已经算过了，
每次都重新算，太浪费！
```

**解决方案**:
```
KV Cache:
把每层的Key和Value缓存起来，
下次只算新token的K/V，
然后和缓存的拼接

效果:
生成速度从O(N²)降到O(N)
长文本生成速度大幅提升
```

## 6.5 LLM的应用

### 提示工程（Prompt Engineering）

**核心技巧**:
```
1. 清晰具体
   ❌ "写个故事"
   ✅ "写一个300字的科幻短篇，主题是AI觉醒"

2. 提供上下文
   "你是一个经验丰富的Python程序员..."

3. Few-shot示例
   "把以下英文翻译成中文:
    Hello → 你好
    World → 世界
    Machine Learning → ?"

4. 思维链（Chain of Thought）
   "解这道题，请一步步思考..."

5. 结构化输出
   "以JSON格式返回结果..."
```

### RAG（检索增强生成）

**问题**: LLM的知识有限，容易"幻觉"

**解决方案**:
```
用户提问 → 向量化 → 检索相关知识 → 
拼接上下文 → LLM生成回答

好处:
- 减少幻觉
- 知识可更新
- 可溯源
```

### Agent（智能体）

**概念**: 让LLM能够使用工具、与环境交互

```
架构:
用户请求 → LLM思考 → 决定行动 → 执行工具 → 
观察结果 → LLM再思考 → ... → 给出答案

工具类型:
├── 搜索引擎
├── 计算器
├── 代码执行器
├── 数据库查询
└── API调用

ReAct模式:
Thought（思考）→ Action（行动）→ Observation（观察）
循环直到得到答案
```

## 6.6 本地部署LLM

### 模型选择（适合你的电脑）

| 模型 | 参数 | 量化后 | 速度 | 能力 |
|------|------|--------|------|------|
| **TinyLlama** | 1.1B | 600MB | 很快 | 基础对话 |
| **Phi-3 Mini** | 3.8B | 2.3GB | 快 | 强推理 |
| **Qwen2.5-3B** | 3B | 2GB | 快 | 中文好 |
| **Llama 3.2 3B** | 3B | 2GB | 快 | 均衡 |
| **Gemma 2B** | 2B | 1.5GB | 很快 | 安全 |

### 部署工具

**Ollama（推荐）**:
```bash
# 安装后使用
ollama run qwen:4b      # 运行千问4B
ollama run llama3.2:3b  # 运行Llama 3.2 3B

# REST API调用
curl http://localhost:11434/api/generate -d '{
  "model": "qwen:4b",
  "prompt": "你好"
}'
```

**llama.cpp**:
```bash
# 直接运行GGUF模型
./main -m model.q4_0.gguf -c 2048 -p "你好"

# 特点:
- 纯C++实现
- CPU推理最快
- 支持AVX2优化
```

**Python推理**:
```python
from transformers import AutoModelForCausalLM, AutoTokenizer

# 加载模型
tokenizer = AutoTokenizer.from_pretrained("Qwen/Qwen2.5-3B-Instruct")
model = AutoModelForCausalLM.from_pretrained(
    "Qwen/Qwen2.5-3B-Instruct",
    device_map="cpu",
    torch_dtype="auto"
)

# 生成
text = "你好"
inputs = tokenizer(text, return_tensors="pt")
outputs = model.generate(**inputs, max_new_tokens=100)
print(tokenizer.decode(outputs[0]))
```

### 实验3：简单的RAG实现

```python
from sentence_transformers import SentenceTransformer
import numpy as np

# 1. 准备知识库
documents = [
    "Python是一种解释型编程语言",
    "Java是一种面向对象的编程语言",
    "JavaScript主要用于网页开发",
    "Python的创始人是Guido van Rossum",
    "Python支持多种编程范式"
]

# 2. 加载嵌入模型（轻量级）
embedder = SentenceTransformer('paraphrase-MiniLM-L6-v2')

# 3. 编码知识库
doc_embeddings = embedder.encode(documents)

def search(query, top_k=2):
    # 编码查询
    query_embedding = embedder.encode([query])
    
    # 计算相似度
    similarities = np.dot(doc_embeddings, query_embedding.T).squeeze()
    
    # 返回最相关的文档
    top_indices = np.argsort(similarities)[-top_k:][::-1]
    return [documents[i] for i in top_indices]

# 4. 使用
query = "谁创建了Python？"
relevant_docs = search(query)

print("检索到的相关知识:")
for doc in relevant_docs:
    print(f"  - {doc}")

# 5. 构造Prompt
context = "\n".join(relevant_docs)
prompt = f"基于以下知识回答问题:\n{context}\n\n问题: {query}\n答案:"

print(f"\n构造的Prompt:\n{prompt}")

# 6. 调用LLM生成答案
# （可以用Ollama API或其他方式）
```

## 本章小结

**核心概念**:
1. LLM是参数量巨大的语言模型，有涌现能力
2. Decoder-only架构是主流
3. 训练分三阶段：预训练、SFT、RLHF
4. 推理时采样策略影响输出质量
5. 本地部署需要量化小模型

**关键认知**:
- ✅ LLM的能力随规模非线性增长
- ✅ 预训练学习语言，微调学习对话
- ✅ RLHF/DPO让模型对齐人类偏好
- ✅ 提示工程能大幅提升LLM效果
- ✅ 3B级量化模型可以在你的电脑上运行
