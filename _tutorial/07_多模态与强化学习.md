## 7.1 多模态AI

### 什么是多模态？

**多模态 = 处理和理解多种类型的数据（文本、图像、音频、视频）**

```
人类是多模态的:
├── 眼睛看图像
├── 耳朵听声音
├── 嘴巴说话
└── 大脑统一处理所有信息

传统AI是单模态的:
├── CV模型只看图
├── NLP模型只看文字
└── 语音模型只听声音

多模态AI:
融合多种信息，统一理解
```

### 为什么需要多模态？

```
例1: 看图说话
单看图: 知道有猫、沙发
单看文字: 不知道具体场景
图文结合: "一只橘猫在沙发上睡觉"

例2: 视频理解
图像: 知道每帧有什么
音频: 知道说了什么
结合: 理解完整剧情

例3: 自动驾驶
摄像头: 看到红绿灯
雷达: 测到距离
地图: 知道路线
结合: 做出驾驶决策
```

### 多模态任务

| 任务 | 输入 | 输出 | 例子 |
|------|------|------|------|
| **图像描述** | 图片 | 文字描述 | "一只狗在玩球" |
| **视觉问答** | 图片+问题 | 答案 | "图中有几个人？"→"3个" |
| **图文检索** | 文字 | 相关图片 | 搜"日落"找日落照片 |
| **图像生成** | 文字描述 | 图片 | DALL-E、Midjourney |
| **语音识别** | 音频 | 文字 | 语音转文字 |
| **语音合成** | 文字 | 音频 | 文字转语音 |
| **视频理解** | 视频 | 描述/标签 | 视频内容摘要 |

## 7.2 多模态技术架构

### 核心挑战

```
问题1: 不同模态怎么表示？
图像: 像素矩阵 [H×W×3]
文本: Token序列 [seq_len]
音频: 波形或频谱 [time×freq]
→ 需要统一表示！

问题2: 不同模态怎么对齐？
"狗"这个字 ←→ 狗的图片
需要在同一个空间中表示

问题3: 不同模态怎么融合？
简单拼接？加权求和？注意力机制？
```

### 架构类型

**类型1: 双塔结构**
```
图像编码器 ──┐
             ├──→ 对比学习 → 对齐
文本编码器 ──┘

代表: CLIP

特点:
- 各自编码，最后对齐
- 适合检索任务
- 效率高
```

**类型2: 编码器-解码器**
```
图像编码器 → 文本解码器 → 描述

代表: Show-Attend-Tell

特点:
- 编码器提取特征
- 解码器生成文本
- 适合生成任务
```

**类型3: 统一架构**
```
所有模态 → 统一Transformer → 输出

代表: GPT-4V、Gemini

特点:
- 一个模型处理所有模态
- 能力最强
- 数据、算力要求最高
```

**类型4: 适配器架构**
```
冻结大语言模型 ← 投影层 ← 视觉编码器

代表: LLaVA、MiniGPT-4

特点:
- 利用预训练LLM
- 只训练投影层
- 成本低效果好
```

### CLIP详解

**核心思想**: 对比学习对齐图文

```
训练数据:
4亿对(图像, 文本)从互联网收集

训练目标:
让匹配的图像和文本在向量空间接近
不匹配的远离

损失函数:
对称交叉熵（图像→文本 + 文本→图像）
```

**使用方式**:
```
1. 图像分类（Zero-shot）
   图像 → 向量
   ["猫", "狗", "鸟"] → 向量
   找最相似的

2. 图文检索
   文字搜索相关图片
   图片搜索相关文字

3. 作为其他模型的视觉编码器
```

### LLaVA详解

**架构**:
```
图像 → ViT(视觉Transformer) → 投影层 → 
与文本拼接 → 大语言模型 → 输出
```

**训练**:
```
阶段1: 预训练投影层
- 冻结视觉编码器和LLM
- 只训练投影层
- 用图文对数据

阶段2: 端到端微调
- 冻结视觉编码器
- 微调投影层和LLM
- 用指令跟随数据
```

## 7.3 轻量多模态模型

### 适合你的电脑（8GB RAM）

| 模型 | 视觉 | 语言 | 大小 | 特点 |
|------|------|------|------|------|
| **LLaVA-Phi-3** | CLIP | Phi-3(3.8B) | ~4GB | 轻量好用 |
| **Bunny-Llama-3B** | SigLIP | TinyLlama | ~2GB | 极轻量 |
| **Moondream2** | 专用小模型 | Phi-2(2.7B) | ~1.6GB | 资源友好 |
| **MobileVLM** | 轻量ViT | 轻量LLM | ~3GB | 移动端优化 |

### 使用示例（LLaVA）

```python
from transformers import AutoProcessor, LlavaForConditionalGeneration
from PIL import Image

# 加载模型（首次下载约4GB）
model = LlavaForConditionalGeneration.from_pretrained(
    "llava-hf/llava-phi-3-hf",
    torch_dtype="float16",
    device_map="auto"
)
processor = AutoProcessor.from_pretrained("llava-hf/llava-phi-3-hf")

# 准备输入
image = Image.open("photo.jpg")
prompt = "USER: <image>\n描述这张图片\nASSISTANT:"

# 生成
inputs = processor(text=prompt, images=image, return_tensors="pt")
output = model.generate(**inputs, max_new_tokens=200)
result = processor.decode(output[0], skip_special_tokens=True)
print(result)
```

## 7.4 强化学习（RL）

### 什么是强化学习？

**强化学习 = 通过与环境交互，试错学习最优行为**

```
监督学习:
输入 → [模型] → 输出
↑              ↓
已知正确答案 ←──┘

强化学习:
状态 → [策略] → 动作 → 环境 → 新状态 + 奖励
              ↑                  ↓
              └────── 学习 ──────┘

类比:
训练小狗:
- 做对了 → 给零食（正奖励）
- 做错了 → 批评（负奖励）
- 小狗学会什么该做什么不该做
```

### 核心概念

| 概念 | 符号 | 说明 |
|------|------|------|
| **Agent** | - | 智能体，做决策的主体 |
| **Environment** | - | 环境，Agent交互的对象 |
| **State** | s | 状态，当前情况的描述 |
| **Action** | a | 动作，Agent能做的事 |
| **Reward** | r | 奖励，环境给Agent的反馈 |
| **Policy** | π(a\|s) | 策略，状态到动作的映射 |
| **Value** | V(s) | 价值，从某状态出发的期望累积奖励 |

### 强化学习的特点

```
优点:
├── 不需要标注数据
├── 可以处理序列决策
├── 能够探索未知策略
└── 适合游戏、机器人等交互任务

缺点:
├── 训练慢（需要大量试错）
├── 样本效率低
├── 奖励设计困难
├── 不稳定（同样的设置可能结果不同）
```

## 7.5 强化学习算法

### Q-Learning（价值学习）

**思想**: 学习每个状态-动作对的价值

```
Q(s,a) = 在状态s执行动作a，之后能获得的总奖励期望

学习过程:
1. 在当前状态s，根据Q值选择动作a
2. 执行a，获得奖励r，到达新状态s'
3. 更新Q值:
   Q(s,a) = Q(s,a) + α[r + γmaxQ(s',a') - Q(s,a)]
   
   α: 学习率
   γ: 折扣因子（未来奖励的重要性）
```

**问题**: 状态空间太大时，Q表存不下

**解决**: DQN（Deep Q-Network）
```
用神经网络近似Q函数
Q(s,a) ≈ NeuralNetwork(s)
```

### Policy Gradient（策略梯度）

**思想**: 直接优化策略（而不是学价值）

```
策略π(a|s; θ): 神经网络，参数θ

目标: 最大化期望累积奖励 J(θ)

梯度: ∇J(θ) = E[∇log π(a|s) × 优势函数]

直观理解:
- 如果动作带来了高奖励 → 增加该动作概率
- 如果动作带来了低奖励 → 降低该动作概率
```

### PPO（近端策略优化）

**目前最常用的RL算法**

```
改进点:
普通Policy Gradient更新步长难控制
- 太大: 策略崩溃
- 太小: 收敛慢

PPO解决方案:
用Clipped Surrogate Objective限制更新幅度

Loss = min(r(θ)A, clip(r(θ), 1-ε, 1+ε)A)

r(θ) = 新策略概率 / 旧策略概率
A = 优势函数（这个动作比平均好多少）
ε = 裁剪系数（通常0.1-0.2）

效果:
- 训练更稳定
- 实现简单
- 效果好
```

### RLHF中的RL

```
在LLM训练中:

奖励模型: 给生成的回答打分
策略: 语言模型（生成回答）

训练循环:
1. 语言模型生成回答
2. 奖励模型打分
3. PPO算法更新语言模型
4. 加上KL约束（不要太偏离原模型）

目标:
让语言模型学会生成高分的回答
```

## 7.7 动手试试

### 实验1：用CLIP做图文匹配

```python
import torch
from PIL import Image
from transformers import CLIPProcessor, CLIPModel

# 加载模型（约600MB）
model = CLIPModel.from_pretrained("openai/clip-vit-base-patch32")
processor = CLIPProcessor.from_pretrained("openai/clip-vit-base-patch32")

# 准备图像和文本
image = Image.open("cat.jpg")
texts = ["a photo of a cat", "a photo of a dog", "a photo of a car"]

# 编码
inputs = processor(text=texts, images=image, return_tensors="pt", padding=True)
outputs = model(**inputs)

# 计算相似度
logits_per_image = outputs.logits_per_image
probs = logits_per_image.softmax(dim=1)

# 输出结果
for text, prob in zip(texts, probs[0]):
    print(f"{text}: {prob.item():.4f}")

# 应该 "a photo of a cat" 得分最高
```

### 实验2：简单的强化学习（OpenAI Gym）

```python
import gym
import numpy as np

# 创建环境
env = gym.make('CartPole-v1')

# 随机策略演示
for episode in range(3):
    state = env.reset()
    total_reward = 0
    
    for step in range(100):
        env.render()  # 可视化
        
        # 随机动作
        action = env.action_space.sample()
        
        # 执行动作
        state, reward, done, _ = env.step(action)
        total_reward += reward
        
        if done:
            break
    
    print(f"Episode {episode+1}, Reward: {total_reward}")

env.close()

# 观察:
# 随机策略很快就失败（杆子倒下）
# 训练RL后，可以保持平衡很久
```
