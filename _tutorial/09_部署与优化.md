## 9.1 模型部署方式

### 部署场景对比

| 场景 | 特点 | 技术方案 | 适用 |
|------|------|----------|------|
| **本地部署** | 在自己的电脑/设备上运行 | ONNX Runtime, llama.cpp | 个人使用、隐私要求高 |
| **服务器部署** | 部署到云服务器 | FastAPI, Triton | 对外提供服务 |
| **边缘部署** | 在手机/嵌入式设备运行 | TFLite, ncnn | 移动端APP |
| **浏览器部署** | 直接在浏览器运行 | ONNX.js, TensorFlow.js | 无服务器、隐私保护 |

### 你的部署选择

基于你的电脑配置（8GB RAM，无独立显卡）：

```
推荐方案:
├── 本地开发测试: PyTorch/TensorFlow CPU版本
├── 生产级本地部署: ONNX Runtime
├── LLM本地部署: Ollama/llama.cpp
└── 小模型边缘部署: 转换为TFLite

不推荐:
├── TensorRT（需要NVIDIA GPU）
├── 大模型微调后部署（内存不够）
└── 高并发服务部署（性能不足）
```

## 9.2 模型优化技术

### 1. 模型量化（Quantization）

**什么是量化？**
```
把模型权重从高精度浮点数转为低精度整数
FP32 (32位) → FP16 (16位) → INT8 (8位) → INT4 (4位)

效果:
- 模型大小减小 2-8倍
- 推理速度提升 2-4倍
- 内存占用降低
- 精度损失通常很小（1-3%）
```

**PyTorch量化**
```python
import torch.quantization

# 动态量化（最简单）
model_quantized = torch.quantization.quantize_dynamic(
    model, 
    {torch.nn.Linear},  # 要量化的层
    dtype=torch.qint8
)

# 保存
torch.save(model_quantized.state_dict(), 'model_quantized.pth')

# 大小对比：
# 原始模型: 100MB
# 量化后: 25-30MB
```

**ONNX量化**
```python
from onnxruntime.quantization import quantize_dynamic, QuantType
import onnx

# 导出ONNX模型后量化
quantize_dynamic(
    model_input='model.onnx',
    model_output='model_quantized.onnx',
    weight_type=QuantType.QInt8
)
```

**GGUF量化（LLM专用）**
```
Q4_0:  4-bit,  fastest, slightly lower quality
Q4_K_M: 4-bit, balanced, recommended
Q5_K_M: 5-bit, better quality
Q8_0:  8-bit,  near-original quality

命令示例:
python convert.py --input model.pth --output model.gguf --quantize Q4_K_M
```

### 2. 模型剪枝（Pruning）

**什么是剪枝？**
```
移除模型中不重要或冗余的权重
把接近0的权重直接设为0

效果:
- 稀疏模型，可以压缩存储
- 配合硬件加速，可提升速度
- 需要专门的稀疏计算库
```

**简单示例**
```python
import torch.nn.utils.prune as prune

# 对某层进行剪枝
prune.l1_unstructured(
    model.fc1, 
    name='weight', 
    amount=0.3  # 剪掉30%的权重
)

# 使剪枝永久化
prune.remove(model.fc1, 'weight')
```

### 3. 知识蒸馏（Knowledge Distillation）

**什么是蒸馏？**
```
用大模型（教师）教小模型（学生）
让小模型学会大模型的"软标签"

为什么有效？
大模型的概率分布包含更多信息
比如: 狗(0.7), 狼(0.2), 狐狸(0.1)
比硬标签 "狗(1.0)" 包含更多相似性信息
```

**蒸馏流程**
```python
# 教师模型（大，已训练好）
teacher_model = load_large_model()
teacher_model.eval()

# 学生模型（小，待训练）
student_model = SmallModel()

for data, target in train_loader:
    # 教师预测
    with torch.no_grad():
        teacher_logits = teacher_model(data)
    
    # 学生预测
    student_logits = student_model(data)
    
    # 蒸馏损失（学生模仿教师）
    distillation_loss = nn.KLDivLoss()(
        F.log_softmax(student_logits / T, dim=1),
        F.softmax(teacher_logits / T, dim=1)
    ) * (T * T)
    
    # 学生损失（学生也要对真实标签正确）
    student_loss = criterion(student_logits, target)
    
    # 总损失
    loss = alpha * student_loss + (1 - alpha) * distillation_loss
```

### 4. 模型转换

**PyTorch → ONNX**
```python
import torch.onnx

# 创建示例输入
dummy_input = torch.randn(1, 3, 224, 224)

# 导出
torch.onnx.export(
    model,
    dummy_input,
    'model.onnx',
    input_names=['input'],
    output_names=['output'],
    dynamic_axes={'input': {0: 'batch_size'}, 'output': {0: 'batch_size'}}
)
```

**ONNX → TensorFlow Lite**
```bash
# 先转 SavedModel
onnx-tf convert -i model.onnx -o saved_model

# 再转 TFLite
tflite_convert --saved_model_dir=saved_model --output_file=model.tflite
```

## 9.3 本地部署实践

### 方案1：ONNX Runtime（推荐）

**为什么选ONNX Runtime？**
```
优势:
├── 跨平台（Windows/Linux/Mac）
├── 支持CPU优化
├── 推理速度快
├── 支持多种深度学习框架
└── 支持量化模型
```

**部署代码**
```python
import onnxruntime as ort
import numpy as np
from PIL import Image

# 加载模型
session = ort.InferenceSession('model_quantized.onnx')

# 获取输入输出信息
input_name = session.get_inputs()[0].name
output_name = session.get_outputs()[0].name

# 准备输入
img = Image.open('cat.jpg')
img_array = preprocess(img)  # 你的预处理函数
img_array = np.expand_dims(img_array, axis=0).astype(np.float32)

# 推理
outputs = session.run([output_name], {input_name: img_array})
predictions = outputs[0]

# 后处理
class_idx = np.argmax(predictions)
print(f"预测类别: {class_idx}")
```

### 方案3：Python服务化（FastAPI）

**创建API服务**
```python
from fastapi import FastAPI, File, UploadFile
from fastapi.responses import JSONResponse
import uvicorn
import onnxruntime as ort
from PIL import Image
import io
import numpy as np

app = FastAPI(title="图像分类API")

# 加载模型（启动时加载）
session = ort.InferenceSession('model.onnx')
input_name = session.get_inputs()[0].name
output_name = session.get_outputs()[0].name

def preprocess(image):
    """图像预处理"""
    image = image.resize((224, 224))
    image = np.array(image).astype(np.float32) / 255.0
    image = (image - [0.485, 0.456, 0.406]) / [0.229, 0.224, 0.225]
    image = np.transpose(image, (2, 0, 1))
    return np.expand_dims(image, axis=0)

@app.post("/predict")
async def predict(file: UploadFile = File(...)):
    # 读取图片
    contents = await file.read()
    image = Image.open(io.BytesIO(contents)).convert('RGB')
    
    # 预处理
    input_data = preprocess(image)
    
    # 推理
    outputs = session.run([output_name], {input_name: input_data})
    predictions = outputs[0][0]
    
    # 返回结果
    class_idx = int(np.argmax(predictions))
    confidence = float(predictions[class_idx])
    
    return JSONResponse({
        'class': class_idx,
        'confidence': confidence,
        'all_probabilities': predictions.tolist()
    })

@app.get("/health")
async def health():
    return {'status': 'healthy'}

if __name__ == '__main__':
    uvicorn.run(app, host='0.0.0.0', port=8000)
```

**启动与测试**

```bash
python api.py

# 测试
```

## 9.4 性能优化技巧

### 批处理推理

```python
# 单条推理（慢）
for img in images:
    result = model(img)

# 批处理（快，充分利用并行计算）
batch_size = 32
for i in range(0, len(images), batch_size):
    batch = images[i:i+batch_size]
    results = model(batch)
```

### 异步处理

```python
import asyncio
import concurrent.futures

# 使用线程池处理CPU密集型推理
executor = concurrent.futures.ThreadPoolExecutor(max_workers=4)

async def predict_async(image):
    loop = asyncio.get_event_loop()
    result = await loop.run_in_executor(executor, model.predict, image)
    return result

# FastAPI中
@app.post("/predict")
async def predict(file: UploadFile = File(...)):
    image = await load_image(file)
    result = await predict_async(image)
    return result
```

### 模型预热与缓存

```python
# 预热：先跑几个假数据，让模型加载到内存/缓存
for _ in range(3):
    dummy_input = np.random.randn(1, 3, 224, 224).astype(np.float32)
    session.run([output_name], {input_name: dummy_input})

# 单例模式保持模型常驻内存
class ModelSingleton:
    _instance = None
    
    @classmethod

        if cls._instance is None:
```

## 9.5 监控与维护

### 日志记录

```python
import logging
import time

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

@app.post("/predict")
async def predict(file: UploadFile = File(...)):
    start_time = time.time()
    
    try:
        # 处理...
        result = process(file)
        
        # 记录成功
        logger.info(f"Prediction success, time: {time.time() - start_time:.3f}s")
        return result
        
    except Exception as e:
        # 记录错误
        logger.error(f"Prediction failed: {str(e)}")
        return {'error': str(e)}
```

### 性能监控

```python
import psutil
import time

class PerformanceMonitor:
    def __init__(self):
        self.request_times = []
        
    def log_request(self, duration):
        self.request_times.append(duration)
        if len(self.request_times) > 1000:
            self.request_times = self.request_times[-1000:]
    
    def get_stats(self):
        if not self.request_times:
            return {}
        return {
            'avg_latency': sum(self.request_times) / len(self.request_times),
            'max_latency': max(self.request_times),
            'min_latency': min(self.request_times),
            'cpu_percent': psutil.cpu_percent(),
            'memory_percent': psutil.virtual_memory().percent
        }

monitor = PerformanceMonitor()

@app.get("/stats")
async def stats():
    return monitor.get_stats()
```

## 9.6 部署 checklist

```
部署前检查:

□ 模型是否已优化（量化/剪枝）？
□ 是否处理了各种输入边界情况？
□ 错误处理是否完善？
□ 日志是否记录关键信息？
□ 性能是否满足需求？
□ 内存使用是否在限制内？
□ 是否测试过多并发情况？

部署后监控:

□ 服务是否正常启动？
□ 响应时间是否合理？
□ 内存是否稳定增长（泄露）？
□ 错误率是否可接受？
□ 日志是否有异常？
```

## 本章小结

**核心优化技术：**
1. 量化 - 减小模型、提升速度
2. 剪枝 - 移除冗余参数
3. 蒸馏 - 大模型教小模型
4. 转换 - 使用高效推理框架

**部署方案：**
- ONNX Runtime - 通用、高效
- Ollama - LLM专用、简单
- FastAPI - 服务化、可扩展

**优化原则：**
- 先量化，再考虑其他
- 批处理提升吞吐量
- 预热减少冷启动延迟
- 监控保证稳定性
