## 一、机器学习基础

### 1.1 核心概念

**定义**: 让计算机从数据中学习规律，无需显式编程。

```
传统编程: 规则 + 数据 → 答案
机器学习: 数据 + 答案 → 规则(模型)
```

### 1.2 学习范式

| 类型 | 数据 | 目标 | 典型应用 |
|------|------|------|----------|
| **监督学习** | 带标签数据 | 学习输入→输出映射 | 分类、回归 |
| **无监督学习** | 无标签数据 | 发现数据内在结构 | 聚类、降维 |
| **半监督学习** | 少量标签+大量无标签 | 利用未标注数据 | 标注成本高的场景 |
| **自监督学习** | 自动生成标签 | 预训练表示 | LLM预训练 |
| **强化学习** | 环境交互 | 最大化累积奖励 | 游戏、机器人 |

### 1.3 经典算法

**监督学习:**
```
├── 线性回归: y = wx + b
├── 逻辑回归: 二分类基础
├── 决策树: 可解释性强
├── 随机森林: 集成多棵树
├── SVM: 最大化间隔分类
└── 朴素贝叶斯: 概率分类
```

**无监督学习:**
```
├── K-Means: 基于距离的聚类
├── DBSCAN: 密度聚类
├── PCA: 主成分降维
├── t-SNE: 非线性降维可视化
└── GMM: 高斯混合模型
```

### 1.4 模型评估

| 指标 | 适用场景 | 公式/说明 |
|------|----------|-----------|
| **Accuracy** | 平衡数据集 | 正确率 = (TP+TN)/(总样本) |
| **Precision** | 关注误报 | 精确率 = TP/(TP+FP) |
| **Recall** | 关注漏报 | 召回率 = TP/(TP+FN) |
| **F1-Score** | 平衡P和R | 2×P×R/(P+R) |
| **AUC-ROC** | 排序能力 | ROC曲线下面积 |
| **MSE/RMSE** | 回归任务 | 均方误差 |

**交叉验证:**
```
K折交叉验证:
1. 数据分成K份
2. 轮流用K-1份训练，1份验证
3. 取K次平均作为最终评估
```

## 二、深度学习核心

### 2.1 基础组件

**感知机(神经元):**
```
输入 x₁, x₂, ..., xₙ
    ↓
加权求和: z = Σ(wᵢxᵢ) + b
    ↓
激活函数: a = f(z)
    ↓
输出
```

**激活函数:**

| 函数 | 公式 | 特点 | 适用 |
|------|------|------|------|
| **ReLU** | max(0,x) | 计算快、缓解梯度消失 | 隐藏层默认 |
| **Leaky ReLU** | max(αx,x) | 解决Dead ReLU | 深层网络 |
| **Sigmoid** | 1/(1+e⁻ˣ) | 输出0-1 | 二分类输出 |
| **Tanh** | (eˣ-e⁻ˣ)/(eˣ+e⁻ˣ) | 输出-1~1 | RNN隐藏层 |
| **Softmax** | eˣⁱ/Σeˣʲ | 多类概率归一 | 多分类输出 |
| **GELU** | x·Φ(x) | 平滑、BERT/GPT使用 | Transformer |

### 2.2 反向传播

**核心思想**: 链式法则计算梯度，从输出层向输入层传播误差。

```
前向传播: 计算预测值 → 计算损失
    ↓
反向传播: ∂Loss/∂W = ∂Loss/∂Output × ∂Output/∂W
    ↓
参数更新: W = W - lr × ∂Loss/∂W
```

**优化器:**

| 优化器 | 特点 | 适用 |
|--------|------|------|
| **SGD** | 基础、需调学习率 | 简单任务 |
| **Momentum** | 加速收敛、抑制震荡 | CV常用 |
| **Adam** | 自适应学习率、默认首选 | 通用 |
| **AdamW** | 解耦权重衰减 | Transformer |
| **Lion** | 内存省、收敛快 | 大模型训练 |

### 2.3 正则化技术

```
防止过拟合:
├── Dropout: 随机丢弃神经元
├── BatchNorm: 批归一化
├── LayerNorm: 层归一化(Transformer)
├── L1/L2正则: 约束权重大小
├── Early Stopping: 早停
└── Data Augmentation: 数据增强
```

### 2.4 学习率策略

```
学习率调度:
├── Step Decay: 固定步长衰减
├── Cosine Annealing: 余弦退火
├── Warmup: 预热阶段线性增加
├── ReduceLROnPlateau: 根据验证集调整
└── One Cycle: 先升后降
```

## 三、神经网络架构

### 3.1 CNN 卷积神经网络

**核心思想**: 局部连接、权值共享，适合网格结构数据(图像)。

**卷积操作:**
```
输入特征图 (H×W×C)
    ↓
卷积核 (K×K×C×N) 滑动计算
    ↓
输出特征图 (H'×W'×N)
```

**经典架构演进:**

| 年份 | 模型 | 核心创新 | 参数量 |
|------|------|----------|--------|
| 1998 | LeNet | 首个成功CNN | 60K |
| 2012 | AlexNet | ReLU+Dropout+GPU | 60M |
| 2014 | VGG | 小卷积核(3×3)堆叠 | 138M |
| 2015 | ResNet | 残差连接 | 25M-60M |
| 2016 | DenseNet | 密集连接 | - |
| 2019 | EfficientNet | 复合缩放 | 5M-66M |

**常用层:**
```
├── Conv2d: 特征提取
├── MaxPool: 下采样
├── BatchNorm: 加速训练
├── Dropout: 正则化
├── GlobalAvgPool: 替代全连接
└── Fully Connected: 分类
```

### 3.2 RNN 循环神经网络

**核心思想**: 处理序列数据，具有记忆能力。

**基础RNN:**
```
hₜ = tanh(Wₕₕ·hₜ₋₁ + Wₓₕ·xₜ + b)
```

**变体对比:**

| 模型 | 解决问题 | 核心机制 |
|------|----------|----------|
| **LSTM** | 长期依赖 | 门控机制(输入/遗忘/输出门) |
| **GRU** | 简化LSTM | 更新门+重置门 |
| **BiLSTM** | 双向上下文 | 正反两个LSTM |
| **Seq2Seq** | 序列转换 | Encoder-Decoder |
| **Attention** | 长距离依赖 | 注意力权重 |

### 3.3 Transformer 架构

**革命性创新**: 完全基于注意力，并行计算，长距离依赖。

```
输入 → Embedding + Positional Encoding
    ↓
[Encoder] × N
├── Multi-Head Self-Attention
├── Add & Norm
├── Feed Forward
└── Add & Norm
    ↓
[Decoder] × N (可选)
├── Masked Self-Attention
├── Cross-Attention
├── Feed Forward
└── Add & Norm
    ↓
输出
```

**注意力机制:**
```python
# Scaled Dot-Product Attention
Q, K, V = Linear(X), Linear(X), Linear(X)  # 投影
scores = Q @ K.T / sqrt(d_k)               # 计算相似度
weights = Softmax(scores)                  # 归一化
output = weights @ V                       # 加权求和
```

**Multi-Head Attention:**
```
将Q/K/V分成h个头，分别计算注意力，再拼接投影
优势: 不同头学习不同子空间特征
```

## 六、大语言模型(LLM)

### 6.1 核心原理

**基于Transformer Decoder的自回归语言模型:**

```
输入文本 → Token化 → Embedding → Transformer层 × N → 输出概率 → 生成文本
```

| 组件 | 功能 | 说明 |
|------|------|------|
| **Tokenizer** | 文本切分 | BPE/SentencePiece编码 |
| **Embedding** | 向量表示 | 可学习或旋转位置编码 |
| **Attention** | 注意力机制 | 计算词间关联权重 |
| **FFN** | 前馈网络 | SwiGLU等现代变体 |

### 6.2 关键参数

```
模型参数量 = 词表嵌入 + 层数 × (Attention参数量 + FFN参数量)
```

| 规模 | 参数量 | 显存需求(FP16) | 适用场景 |
|------|--------|----------------|----------|
| Tiny | 1B-3B | 2-6 GB | 边缘设备 |
| Small | 3B-7B | 6-14 GB | 本地部署 |
| Medium | 7B-13B | 14-26 GB | 专业任务 |
| Large | 13B-70B | 26GB+ | 云端部署 |
| Huge | 70B+ | 多卡 | 研究级 |

### 6.3 训练阶段

```
预训练(Pre-training):
├── 数据: 海量无标注文本
├── 任务: 下一token预测
├── 成本: 极高(百万美元级)
└── 输出: 基础模型

微调(Fine-tuning):
├── SFT: 指令微调
├── RLHF: 人类反馈强化学习
├── DPO: 直接偏好优化
└── 输出: 对齐模型
```

### 6.4 推理与生成

**采样策略:**
- **Greedy**: 总是选最高概率词
- **Temperature**: 控制随机性
- **Top-p**: 核采样
- **Top-k**: 限制候选词
- **Repetition Penalty**: 抑制重复

**KV-Cache优化:**
```
问题: 每次生成都要重新计算之前token的K/V
方案: 缓存历史K/V，只计算新token
效果: 生成长度增加时速度保持稳定
```

## 七、多模态AI

### 7.1 核心思想

融合多种模态(文本、图像、音频、视频)的信息进行统一理解和生成。

```
单模态 → 特征提取 → 模态对齐 → 联合表示 → 多模态任务
```

### 7.2 架构类型

| 类型 | 说明 | 代表模型 |
|------|------|----------|
| **双塔结构** | 各模态独立编码，投影对齐 | CLIP |
| **编码器-解码器** | 多模态编码，单模态解码 | BLIP |
| **统一架构** | 单一Transformer处理多模态 | GPT-4V、Gemini |
| **适配器架构** | 冻结LLM+轻量适配器 | LLaVA、MiniGPT-4 |

### 7.3 典型模型

**CLIP (对比学习):**
```
图像编码器(ViT) ──┐
                  ├──→ 对比学习 → 对齐空间
文本编码器(Transformer) ──┘

应用: 零样本分类、图像检索、文生图条件
```

**LLaVA (视觉指令微调):**
```
视觉编码器(CLIP ViT) → 投影层 → 语言模型(Vicuna)
训练: 仅训练投影层，冻结其他
```

### 7.4 轻量多模态模型

| 模型 | 视觉 | 语言 | 大小 | 特点 |
|------|------|------|------|------|
| **LLaVA-Phi-3** | CLIP | Phi-3 | ~4GB | 轻量对话 |
| **MobileVLM** | 轻量ViT | 轻量LLM | ~3GB | 移动端优化 |
| **Bunny-Llama** | SigLIP | TinyLlama | ~2GB | 极轻量 |
| **Moondream** | 专用小模型 | Phi-2 | ~1.6GB | 资源友好 |

## 八、强化学习(RL)

### 8.1 核心概念

**Agent-Environment交互:**
```
Agent ──动作(a)──→ Environment
  ↑                 ↓
  └──状态(s')、奖励(r)──┘
```

**目标**: 学习策略π(a|s)，最大化累积奖励。

### 8.2 主要方法

| 类型 | 代表算法 | 特点 | 适用 |
|------|----------|------|------|
| **基于价值** | DQN、Double DQN | 学习Q函数 | 离散动作 |
| **基于策略** | REINFORCE、PPO | 直接优化策略 | 连续动作 |
| **Actor-Critic** | A2C、A3C、SAC | 结合两者 | 通用 |
| **Model-Based** | MuZero、Dreamer | 学习环境模型 | 样本高效 |

### 8.3 PPO (近端策略优化)

LLM训练中RLHF的核心算法:

```
目标: 在策略更新幅度约束下最大化优势

Loss = min(rₜ(θ)Âₜ, clip(rₜ(θ), 1-ε, 1+ε)Âₜ)

其中:
- rₜ(θ) = πθ(a|s) / πθold(a|s)  概率比
- Âₜ = 优势函数估计
- ε = 裁剪系数(通常0.1-0.2)
```

### 8.4 RLHF 流程

```
1. 预训练语言模型
    ↓
2. 收集人类偏好数据
    └── 同一prompt的两个回答，人工选择更好的
    ↓
3. 训练奖励模型(Reward Model)
    └── 学习预测人类偏好
    ↓
4. 用PPO优化策略
    └── 最大化奖励同时保持与原模型接近
```

## 九、模型训练全流程

### 9.1 数据准备

**数据流程:**
```
原始数据 → 清洗 → 标注/处理 → 划分 → 加载器 → 训练
```

**关键原则:**
```
├── 数据质量 > 数据数量
├── 分布一致: 训练/验证/测试同分布
├── 避免泄漏: 验证集不参与任何决策
└── 数据增强: 扩充多样性
```

### 9.2 训练配置

**超参数选择:**

| 参数 | 经验值 | 说明 |
|------|--------|------|
| **Batch Size** | 32-256 | 越大越稳定，需配合学习率 |
| **Learning Rate** | 1e-5 ~ 1e-3 | 大模型用小LR |
| **Epochs** | 根据early stop | 避免过拟合 |
| **Warmup** | 5-10% steps | 稳定训练初期 |
| **Weight Decay** | 0.01-0.1 | L2正则化 |

### 9.3 训练监控

**关键指标:**
```
├── Loss曲线: 下降平滑为佳
├── 学习率: 按计划变化
├── 梯度范数: 检测爆炸/消失
├── 验证指标: 关注泛化
└── 资源使用: GPU/CPU/内存
```

**调试技巧:**
```
训练不收敛:
├── 检查学习率(太大/太小)
├── 检查数据归一化
├── 检查标签是否正确
└── 检查梯度是否回传

验证集表现差:
├── 增加正则化
├── 增加数据
├── 降低模型复杂度
└── 检查数据泄漏
```

### 9.4 保存与恢复

```python
# 检查点保存
torch.save({
    'epoch': epoch,
    'model_state_dict': model.state_dict(),
    'optimizer_state_dict': optimizer.state_dict(),
    'loss': loss,
}, 'checkpoint.pth')

# 恢复训练
checkpoint = torch.load('checkpoint.pth')
model.load_state_dict(checkpoint['model_state_dict'])
optimizer.load_state_dict(checkpoint['optimizer_state_dict'])
```

## 十、模型优化与部署

### 10.1 模型压缩

| 技术 | 原理 | 效果 | 适用 |
|------|------|------|------|
| **量化** | 降低数值精度 | 2-4x压缩 | 所有模型 |
| **剪枝** | 移除不重要的权重 | 2-10x压缩 | CNN、FC |
| **蒸馏** | 大模型教小模型 | 精度保持 | 分类、检测 |
| **结构化压缩** | 通道/层裁剪 | 推理加速 | CNN |

### 10.2 量化详解

```
FP32 (32位) → FP16 (16位) → INT8 (8位) → INT4 (4位)
  4字节        2字节         1字节         0.5字节
```

**GGUF格式(用于llama.cpp):**

| 量化类型 | 比特/权重 | 7B模型大小 | 质量损失 | 推荐场景 |
|----------|-----------|------------|----------|----------|
| Q4_0 | 4.5 | 3.5 GB | 中 | 资源极度受限 |
| Q4_K_M | 4.75 | 4.0 GB | 低 | **平衡首选** |
| Q5_K_M | 5.5 | 4.7 GB | 很低 | 质量优先 |
| Q8_0 | 8.5 | 7.0 GB | 几乎无 | 接近原精度 |

**ONNX量化:**
```python
import onnx
from onnxruntime.quantization import quantize_dynamic

quantize_dynamic(
    model_input='model.onnx',
    model_output='model_quant.onnx',
    weight_type=QuantType.QInt8
)
```

### 11.4 性能优化建议

```
内存优化:
├── 使用量化模型 (INT8/INT4)
├── 分批加载数据
├── 及时释放不需要的张量
├── 使用内存映射加载大模型
└── 减少数据副本

CPU优化:
├── 启用AVX2指令集编译
├── 设置合适线程数 (物理核心数)
├── 使用Intel MKL/OpenBLAS
├── 批量推理而非单条
└── 预热模型保持缓存

速度优化:
├── 使用ONNX转换
├── 图优化(fuse操作)
├── 算子优化(自定义kernel)
└── 缓存中间结果
```
