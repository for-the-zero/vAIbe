## 10.1 你能指挥我做什么？

### 基于这套教程，你可以让我：

```
✅ 技术方案设计
   └── 根据你的需求，设计完整的技术方案
   
✅ 代码实现
   └── 编写训练脚本、数据处理、模型定义等
   
✅ 调试优化
   └── 排查错误、性能优化、超参数调优
   
✅ 部署配置
   └── 模型转换、API封装、部署脚本
   
✅ 知识讲解
   └── 解释概念、深入某个技术点
```

### 你不能让我做的：

```
❌ 直接训练大模型（>1B参数）
   └── 你的电脑资源不够，我也无法控制云端资源
   
❌ 生成或修改数据集
   └── 你需要自己准备数据
   
❌ 运行耗时很长的训练
   └── 训练时间超过30分钟的任务可能需要分步进行
   
❌ 访问你的本地文件系统（部分限制）
   └── 但你可以复制粘贴内容给我
```

## 10.2 如何描述你的需求

### 好的需求描述 = 成功的一半

**❌ 不好的描述：**
```
"帮我做个AI"
"写个深度学习的代码"
"我想用神经网络"
```

**✅ 好的描述：**
```
"我想做一个能识别猫狗的图片分类器，
我有1000张训练图片，
希望在我的笔记本电脑上运行（8GB内存），
准确率能达到90%以上"
```

### 需求描述模板

```
1. 项目目标
   └── 你想实现什么功能？
   
2. 输入数据
   └── 有什么数据？格式是什么？数量多少？
   
3. 输出要求
   └── 希望得到什么结果？
   
4. 约束条件
   └── 硬件限制？时间限制？精度要求？
   
5. 技术偏好（可选）
   └── 想用什么框架？有什么参考？
```

## 10.3 项目管理流程

### 推荐的协作流程

```
阶段1: 需求确认
├── 你：描述项目目标、数据、约束
├── 我：技术方案、可行性评估
└── 讨论：确定最终方案

阶段2: 环境准备
├── 我：提供环境配置清单、依赖文件
├── 你：安装环境、准备数据
└── 确认：环境运行正常

阶段3: 代码开发
├── 我：编写核心代码（分模块交付）
├── 你：运行测试、反馈问题
└── 迭代：修复bug、优化效果

阶段4: 训练调优
├── 你：运行训练（可能需要多次）
├── 我：分析结果、建议调优
└── 迭代：直到效果满意

阶段5: 部署交付
├── 我：提供部署代码和文档
├── 你：本地部署测试
└── 确认：项目完成
```

### 如何有效沟通

**提供必要信息：**
```
遇到错误时，提供：
├── 完整的错误信息（复制粘贴）
├── 你的代码（相关部分）
├── 你的数据样本（几条示例）
└── 你已经尝试的解决方法
```

**分步进行：**
```
不要把所有需求一次性说完
建议:
├── 先实现最小可用版本(MVP)
├── 验证核心功能可用
├── 再逐步添加功能
└── 这样更容易定位和解决问题
```

**明确反馈：**
```
❌ "有点问题"
✅ "运行到第15行时报错：IndexError: list index out of range"

❌ "效果不太好"
✅ "训练集准确率95%，但验证集只有70%，看起来过拟合了"
```

## 10.4 技术决策指南

### 问题1：选什么模型架构？

**根据任务类型选择：**

| 任务 | 推荐架构 | 理由 |
|------|----------|------|
| 图像分类 | ResNet/MobileNet/EfficientNet | 成熟、效果好 |
| 目标检测 | YOLOv8 | 快、准、易用 |
| 图像分割 | U-Net | 经典、效果好 |
| 文本分类 | BERT/DistilBERT | 理解能力强 |
| 文本生成 | GPT系列 | 生成质量高 |
| 序列预测 | LSTM/Transformer | 捕捉时序关系 |

**根据资源限制选择：**

```
资源极少（手机/嵌入式）:
├── MobileNetV3 (图像)
├── DistilBERT (文本)
├── TinyLlama 1.1B (LLM)
└── 必须量化(INT8)

资源有限（你的电脑）:
├── ResNet18/34 (图像)
├── DistilBERT (文本)
├── Phi-3 Mini 3.8B (LLM)
└── 建议量化(Q4/Q5)

资源充足（云服务器）:
├── ResNet50/101 (图像)
├── BERT-base/large (文本)
├── Llama 3 70B (LLM)
└── FP16/BF16精度
```

### 问题2：从头训练还是微调？

**决策树：**
```
你有预训练模型可用吗？
├── 否 → 从头训练
└── 是 → 你的数据量大吗？
    ├── 大（>10万）→ 可以考虑从头
    └── 小（<1万）→ 强烈建议微调

微调策略：
├── 数据很少（<1000）→ 冻结骨干，只训分类头
├── 数据中等（1000-10000）→ 冻结大部分，微调高层
└── 数据较多（>10000）→ 全部微调，用小学习率
```

### 问题3：如何解决效果不好的问题？

**诊断流程：**
```
训练不收敛？
├── 学习率太大/太小 → 调整学习率
├── 数据未归一化 → 检查预处理
├── 标签错误 → 检查数据
└── 梯度消失/爆炸 → 换激活函数、加BN

过拟合？
├── 训练集准确率↑ 验证集准确率↓
├── 增加数据
├── 数据增强
├── 增加正则化（Dropout、L2）
└── 早停

欠拟合？
├── 训练集和验证集准确率都低
├── 增加模型复杂度
├── 减少正则化
└── 增加训练时间

泛化差？
├── 训练集好，测试集差
├── 检查数据泄漏
├── 检查测试集分布
└── 增加数据多样性
```

### 问题4：推理太慢怎么办？

**优化优先级：**
```
1. 模型量化（最高性价比）
   └── FP32 → INT8，速度提升2-4倍

2. 批处理
   └── 单条→批量，充分利用并行

3. 模型架构优化
   └── 换轻量模型（MobileNet、TinyBERT等）

4. 硬件优化
   └── 使用ONNX Runtime、OpenVINO等

5. 蒸馏
   └── 大模型→小模型（需要额外训练）
```

## 10.7 最佳实践总结

### DO's（要做的）

```
✓ 从小做起，先实现MVP
✓ 使用预训练模型
✓ 及时测试和反馈
✓ 保存检查点
✓ 记录实验结果
✓ 模块化代码结构
✓ 写清楚的数据和约束
```

### DON'Ts（不要做的）

```
✗ 一开始就追求100%完美
✗ 忽视数据质量
✗ 训练集和测试集混用
✗ 过度调参
✗ 一次改太多东西
✗ 不备份代码和模型
```

```
  1. 告诉我训练集/验证集指标

  1. 告诉我当前速度和期望速度
```
